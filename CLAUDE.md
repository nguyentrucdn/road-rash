# CLAUDE.md — Road Rash Web Game

## Project Overview

Browser-based motorcycle combat racing game inspired by the classic Road Rash series. Three.js + TypeScript + Vite. Arcade mode only — pick a track, race 5 AI opponents, fight for position.

## Commands

```bash
npm run dev          # Start dev server
npm run build        # Generate atlas + compile + bundle
npm test             # Run all tests (vitest)
npm run test:watch   # Watch mode tests
npm run generate-atlas  # Regenerate texture atlas only
```

## Architecture

### Game Loop
Fixed 60fps physics tick (`core/GameLoop.ts`) decoupled from render frame rate. Update order: input → AI → physics → combat → road generation → effects → render.

### State Machine
`main.ts` contains the `Game` class managing states: `menu → racing → results`. `startRace(track)` builds the entire scene. `finishRace()` shows results.

### Key Patterns
- **Factory classes** for creating meshes: `VehicleFactory`, `SceneryFactory`, `BikeModelFactory`
- **Entity-based** — `Bike` is the base with physics, `PlayerBike` and `AiBike` wrap it with controllers
- **LOD system** — `SceneryLODManager` swaps detailed meshes for billboards at distance
- **Weather state machine** — `WeatherSystem` with per-track configs, mid-race transitions
- **Texture atlas** — single 2048x2048 PNG generated at build time, loaded once at runtime

### File Organization
- `src/core/` — GameLoop, InputManager, AudioManager
- `src/entities/` — Bike, PlayerBike, AiBike, Weapon
- `src/combat/` — CombatSystem, AttackTypes
- `src/world/` — Road, Environment, TrafficManager
- `src/rendering/` — All visual factories and renderers
- `src/effects/` — SpeedEffects, CombatEffects, NitroEffects, WeatherSystem
- `src/ui/` — HUD, menus, touch controls
- `src/tracks/` — 6 track data definitions
- `src/utils/` — MathUtils, ObjectPool

## Coding Conventions

- All Three.js materials use `flatShading: true` for the low-poly aesthetic
- Vehicle models use realistic PBR values (metalness/roughness) for car paint, glass, chrome, rubber
- Road segments are 5m long (`SEGMENT_LENGTH` in `RoadSegment.ts`)
- Cumulative curve/hill values in Road.ts: `curve * 0.3` lateral, `hill * 0.2` vertical per segment
- Track weather configs live in `effects/WeatherConfig.ts`, lighting in `rendering/LightingManager.ts`
- Tests use vitest with `@/` path alias (configured in `vite.config.ts`)

## Testing

77 tests across 18 files. Key test areas:
- `tests/utils/` — MathUtils, ObjectPool
- `tests/core/` — GameLoop, InputManager
- `tests/entities/` — Bike physics, AiBike behavior
- `tests/combat/` — AttackTypes, CombatSystem hit resolution
- `tests/rendering/` — TextureAtlas, VehicleFactory, SceneryFactory, RoadRenderer, BikeModelFactory
- `tests/effects/` — WeatherConfig, WeatherSystem

Run a specific test: `npx vitest run tests/combat/CombatSystem.test.ts`

## Important Notes

- The texture atlas (`public/textures/atlas.png`) is generated by `scripts/generate-atlas.ts` and committed to git. Only regenerate when textures change (`npm run generate-atlas`).
- The `canvas` npm package (native dependency) is required for atlas generation but not for runtime.
- Three.js chunk size warning during build is expected and not an error.
- `tsconfig.json` uses `"ignoreDeprecations": "6.0"` for TypeScript 6.0 compatibility with the `baseUrl` option.
